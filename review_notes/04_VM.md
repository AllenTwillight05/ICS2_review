## 核心模型

虚拟内存（Virtual Memory, VM）让每个进程看到自己的虚拟地址空间（virtual address space）。CPU 产生虚拟地址（virtual address, VA），MMU 通过页表（page table）把 VA 翻译成物理地址（physical address, PA）。

地址分解：

```text
VA = VPN | VPO
PA = PPN | PPO
```

页大小为 `2^p` bytes 时：

```text
VPO bits = PPO bits = p
VPN bits = VA bits - p
PPN bits = PA bits - p
```

页偏移不变：translation 只把 VPN 变成 PPN，offset 原样保留。

## 题型 1：位宽 fundamentals

QUIZ 典型配置：

- VA = 48 bits。
- PA = 52 bits。
- page size = 4 KB = `2^12` bytes。
- PTE = 8 bytes。
- page-table page = 4 KB。
- L1 data cache = 64 sets, 8-way, 64-byte lines。

计算：

```text
VPO / PPO = 12
VPN = 48 - 12 = 36
PPN = 52 - 12 = 40
entries per page-table page = 4096 / 8 = 512
index bits per level = log2(512) = 9
CO = log2(64) = 6
CI = log2(64 sets) = 6
CT = 52 - CI - CO = 52 - 6 - 6 = 40
```

## 题型 2：多级页表占多少内存

核心原则：count page-table pages, not total mapped pages。

对 x86-64 四级页表：

```text
L1 PML4: bits 47-39
L2 PDPT: bits 38-30
L3 PD:   bits 29-21
L4 PT:   bits 20-12
VPO:     bits 11-0
```

每一级 index 9 bits，每个 page-table page 有 512 entries。

做题步骤：

1. 计算映射区域起始地址和结束地址。
2. 分别拆出 L1/L2/L3/L4 indices。
3. 看每一级有多少不同 index 被使用。
4. 每个被指向的下一级 table 需要一个 4KB page。
5. 总页表内存 = page-table pages 总数 * 4 KB。

QUIZ 例子：映射 8 MB，从 `0x0000_0000_0040_0000` 到 `0x0000_0000_00BF_FFFF`。

4KB regular pages 时：

- L1 index `{0}`，需要 1 个 L1 page。
- L2 index `{0}`，需要 1 个 L2 page。
- L3 index `{2,3,4,5}`，这些 entries 在同一个 L3 page 中，所以需要 1 个 L3 page。
- 每个 2MB 区间需要一个 L4 PT page，共 4 个 L4 pages。
- 总计 `1 + 1 + 1 + 4 = 7 pages = 28 KB`。

常见陷阱：不要把每个虚拟页都算成一个页表页。L4 一个 page-table page 可以覆盖 512 个 4KB pages，也就是 2MB 虚拟地址范围。

## 题型 3：Huge pages

x86-64 的 2 MB huge page 可以让 L3（PD）entry 设置 PS（Page Size）bit，直接映射 2MB physical frame，省掉 L4 page table。

同样的 8MB 映射：

- L1: 1 page。
- L2: 1 page。
- L3: 1 page。
- L4: 0 page。
- 总计 `3 pages = 12 KB`。
- 相比 regular pages 节省 `28 - 12 = 16 KB`。

标准解释：huge pages 不是消灭所有页表开销，而是 prune 掉 L4 level；L1/L2/L3 仍然存在。

附带好处：TLB reach 变大，一个 TLB entry 覆盖 2MB，而不是 4KB。

## 题型 4：VIPT 条件

VIPT（Virtually Indexed, Physically Tagged）：

- 用虚拟地址中的 index bits 先选 cache set。
- 同时 TLB 翻译 VPN 到 PPN。
- 翻译后用 physical tag 做 tag check。

关键条件：

```text
CI + CO <= VPO
```

含义：cache index bits 和 cache offset bits 必须完全落在 page offset 内，因为 page offset 在 VA 和 PA 中相同。

例：32KB L1，64 sets，8-way，64B line，4KB page。

```text
CO = log2(64) = 6
CI = log2(64) = 6
VPO = log2(4096) = 12
CI + CO = 12 <= 12
```

所以 VIPT feasible。

设计 A：64KB L1 = 128 sets * 8-way * 64B。

```text
CI = log2(128) = 7
CO = 6
CI + CO = 13 > 12
```

VIPT breaks。最小 page size 需要 `2^13 = 8 KB`。

设计 B：64KB L1 = 64 sets * 16-way * 64B。

```text
CI = 6
CO = 6
CI + CO = 12 <= 12
```

VIPT still works。

常见陷阱：增大 cache size 不一定破坏 VIPT；增加 sets 会增加 CI，增加 associativity 不增加 CI。

## 题型 5：TLB、page hit、page fault

TLB（Translation Lookaside Buffer）是 MMU 中的小型地址翻译 cache，缓存 VPN -> PPN / PTE 信息。

Page hit 流程：

1. CPU 发出 VA。
2. MMU 查页表或 TLB，发现 PTE valid。
3. 得到 PA。
4. cache/memory 返回数据。

TLB hit：省掉访问内存取 PTE 的一次开销。  
TLB miss：需要访问页表拿 PTE，通常多一次内存访问。

Page fault：

1. CPU 发 VA。
2. MMU 查 PTE，valid bit = 0。
3. 触发 page fault exception。
4. OS handler 判断是否合法。
5. 若合法但页面不在内存，从 disk/page cache/swap 读入，必要时选择 victim page。
6. 若 victim dirty，先写回。
7. 更新 PTE。
8. 返回原进程，重新执行 faulting instruction。

三种情况：

- 合法页不在内存：page in / swap in。
- demand paging：按需加载文件页或 anonymous page。
- 非法访问或权限错误：segmentation fault / SIGSEGV。

## 题型 6：mmap 为什么快

`read + write` 复制文件到 stdout：

```text
disk -> kernel page cache -> user buffer -> kernel output buffer
```

CPU copy 有两次：

1. `read()`：kernel page cache 到 user buffer。
2. `write()`：user buffer 到 kernel output buffer。

`mmap + write`：

```text
disk -> kernel page cache = mapped region -> kernel output buffer
```

`mmap()` 不把文件内容复制到用户 buffer，而是安装 PTE，让用户虚拟地址直接映射 kernel page cache 中的物理页。CPU copy 只剩 `write()` 那一次。

如果只读一个 1GB 文件的前 4KB header：

- `read(fd, buf, 1GB)` 会把整个 1GB 读入并复制到 user buffer。
- `mmap` 是 lazy / demand paging，只有实际触碰的页面才 page fault；只访问 header 通常只读入 4KB。

标准句式：`mmap` wins by copy elimination and demand paging, not simply because it avoids syscalls。

## 题型 7：COW

COW（Copy-on-Write）用于 fork 和 private mapping：

- fork 后父子进程初始共享相同物理页。
- PTE 标为 read-only / private COW。
- 任一进程写该页时触发 page fault。
- fault handler 分配新 physical page，复制原内容，更新写入进程的 PTE。
- 之后双方各自拥有不同副本。

好处：如果 fork 后马上 exec，很多页根本不用复制。

## 题型 8：Page replacement

LRU（Least Recently Used）理想策略：替换最久未访问页面。缺点是精确维护访问顺序开销大。

Clock algorithm：LRU 的近似实现。

- 每个 page 有 reference bit。
- 指针像钟表一样扫描。
- 若 reference bit = 1，清零并跳过。
- 若 reference bit = 0，选为 victim。

答简答时强调：clock 用硬件/OS 维护的访问位近似 LRU，避免精确排序的高开销。

## 动态内存分配简表

`malloc` / `free` 属于 explicit allocator：

- `malloc(size)` 返回至少 `size` bytes 的块，通常对齐。
- `free(p)` 要求 `p` 来自之前的 `malloc/calloc/realloc`，否则 undefined behavior。

Allocator 题如果出现：

- block header：记录 block size 和 allocated bit。
- payload：用户可用区域。
- internal fragmentation：分配块大小超过 payload 需求的浪费。
- explicit free list：空闲块用链表连起来。
- segregated free list：按大小类别分多个 free lists。
- buddy system：物理页分配常用，按 2 的幂 split / merge。

## 高频英文术语

- Virtual address (VA)：虚拟地址。
- Physical address (PA)：物理地址。
- VPN / VPO：virtual page number / virtual page offset。
- PPN / PPO：physical page number / physical page offset。
- Page table：页表。
- PTE: Page Table Entry，页表项。
- MMU: Memory Management Unit，内存管理单元。
- TLB: Translation Lookaside Buffer，地址翻译缓存。
- Page fault：缺页异常。
- Demand paging：按需分页。
- Segmentation fault / SIGSEGV：段错误。
- Multi-level page table：多级页表。
- Huge page：大页。
- VIPT: Virtually Indexed, Physically Tagged。
- mmap: memory mapping，内存映射。
- COW: Copy-on-Write，写时复制。
- Working set：工作集。
- Clock algorithm：时钟替换算法。
